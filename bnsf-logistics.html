Phase 2: Expansion Sprints (Ongoing)

Technical Deep Dive

Architecture Considerations

Data Requirements:

  • Historical shipment data (origin, destination, product, mode selected, outcome)
  • Carload-specific variables (car type, capacity, availability, track access)
  • Shipper profiles (cost sensitivity, ESG priority, service requirements)
  • External data (fuel prices, weather, port congestion, tender rejection rates)

Model Approach:

  • Multi-class classification: Given inputs, recommend optimal mode
  • Probabilistic outputs: "87% confidence boxcar is optimal" with reasoning
  • Feedback loop: Track actual outcomes vs. predictions to improve model
  • Explainability: Must show WHY it recommends boxcar, not just that it does

Integration Points:

  • Input from shipper's TMS/ERP for shipment requirements
  • Output to BNSF systems for execution
  • API for real-time recommendations
  • Dashboard for trend analysis and model performance

Platform Options:

  • Cloud-native (AWS/Azure/GCP) for scalability
  • Containerized for portability
  • API-first for integration flexibility
  • Role-based access for shipper vs. BNSF views
